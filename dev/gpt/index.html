GPT

A small GPT-style language model built on the Autograd engine, the Adam optimizer and the Tokenizer. Each parameter matrix is a single AValue node.

Configuration

A model is described by a GPTConfig: the vocabulary size, embedding width, number of attention heads, number of transformer layers and the maximum sequence length (block_size). head_dim gives the per-head width, n_embd ÷ n_head.

using MicroGPT

docs = ["emma", "olivia", "ava", "isabella", "sophia"]
tok = Tokenizer(docs)

config = GPTConfig(;
    vocab_size=length(tok.uchars) + 1,
    n_embd=16,
    n_head=2,
    n_layer=2,
    block_size=16,
)
head_dim(config)
8

Building a model

GPT builds a model with randomly initialised weights bound to a Tokenizer. It holds the config, a state_dict mapping weight names to matrices, a flat params list for the optimizer, and the tokenizer:

model = GPT(config, tok)
length(model.params)
15

Training

train! trains the model in place on a list of documents with a single Adam optimizer over all weight matrices. Each step forwards one document, accumulates the per-position cross-entropy loss, backpropagates with backward! and takes one step!:

train!(model, docs; num_steps=1000, learning_rate=0.01)

The use_tape keyword selects how backward! finds its topological order: with the tape (the default) the backward pass is a flat reverse walk over the recorded operations instead of a recursive sort of the graph.

Generating

generate samples one sequence from the model and decodes it back to a string, stopping at the BOS token or block_size. temperature controls how sharp the sampling distribution is:

generate(model; temperature=0.5)

Saving and loading

save_model saves the config, the raw weight matrices and the tokenizer vocabulary to a file. load_model reconstructs an equivalent GPT, rebuilding the tokenizer and wrapping each weight matrix back into an AValue:

save_model("model.jls", model)
model = load_model("model.jls")
